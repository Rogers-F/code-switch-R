# ADR-0001: 前端重构方式（保留 Vue3 还是迁移 React）

**状态**：Accepted
**日期**：2026-01-13
**决策日期**：2026-01-13  

---

## 背景

ghosxy 的前端采用 React + shadcn UI + Tailwind；本项目现状为 Vue3 + Tailwind（并已有一定规模的页面与业务逻辑）。

本 ADR 需要决定：UI 2.0 是“保留 Vue3，复刻 ghosxy 的设计语言”，还是“迁移到 React 以最大化复用 ghosxy 组件模式”。

---

## 选项

### 方案 A：保留 Vue3（推荐）

**优点**：

- 变更面更小，向后兼容风险更低
- 复用现有页面逻辑、i18n、Wails 绑定调用方式
- 只需重构 UI 层组件与布局

**缺点**：

- 无法直接复用 ghosxy 的 React 组件代码，需要做“视觉/交互对齐”的二次实现

### 方案 B：迁移到 React

**优点**：

- 可以更贴近 ghosxy 的组件组织方式与生态（shadcn）

**缺点**：

- 成本高：路由、i18n、状态管理、组件全部迁移
- 回归风险大，影响范围远超“UI 重构”

---

## 决策

倾向选择 **方案 A（保留 Vue3）**：以设计对齐为目标，技术栈不对齐；通过建立 Vue 侧的基础组件库来落地 shadcn 风格。

---

## 影响

- P1 需要优先建立一套 Vue 版基础组件（Button/Card/Dialog/ScrollArea/Toast）
- P4 的终端日志视图按 ghosxy 的交互复刻，但不复用代码
