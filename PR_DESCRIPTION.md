# PR 提交申请：全新 macOS 极简高质感 UI 交互升级及多项功能增强与构建优化

## 📖 概述

本 PR 为 Code Switch 项目带来了一次**全方位、大刀阔斧的 UI 视觉重构与开发/构建体验优化**。我们以 **macOS 原生高质感设计语言**为标准，对前端 20+ 个组件和页面进行了细致入微的视觉打磨，彻底统一了设计规范；同时在功能性上，新增了托盘静默设置、WSL 专有网络配置，并设计了全新的交互脚本，极大提升了项目的易用性、可读性与开发编译体验。

---

## 🎨 视觉与交互重构（大白话版：颜值与细节拉满）

我们对原版相对零散的 UI 界面进行了精细化重构，核心改动包括：
1. **全局 macOS 原生设计规范**：
   - **统一按键圆角与尺寸**：全局所有按钮圆角统一调整为 `8px`（R-angle），彻底解决之前按钮大小不一、边缘突兀的问题。
   - **高质感 macOS 强调色**：将全局主操作按钮升级为 macOS 系统经典的 Accent Color，搭配扁平且带柔和微渐变的设计，更显专业和低调。
   - **全局按钮样式收敛**：抽离出全局通用的 `primary-btn` 和 `action-btn` 样式类，完美解决不同页面按钮宽度不一、文字越界的问题。
2. **顶层布局与导航精简**：
   - **告别无意义返回键**：移除顶级视图（如 Availability、Prompts、Skill、SpeedTest、EnvCheck 等）中冗余的“返回”按钮，采用更符合桌面端习惯的侧边栏直达导航。
   - **间距与间距容器统一**：全局重新排版了页面边距（Margins）、模块间距（Spacing）以及 Segment 分段标签控制器，消除页面切换时的视觉跳跃感。
3. **页面功能与细节打磨**：
   - **托盘区交互消除闪烁**：深度优化了禁用托盘弹窗时的窗口切换逻辑，彻底消除点击时“一闪而过”的多余窗口残影。
   - **用量热力图优化**：修复了用量热力图（Usage Heatmap Grid）列宽忽大忽小的问题，恢复固定的列宽与左对齐排列，数据图表更加直观。
   - **精细化可用性仪表盘**：重构了可用性监控列表和速度测试（SpeedTest）页面，移除花哨的 Emoji，改用纯粹专业的高端卡片和指标色，极具科技感。
   - **高级侧边栏与页脚控制**：重构侧边栏，支持用户一键收起/展开，并于底部集成了更加丝滑的主题明暗切换开关。

## ⚡ 实用功能增强（大白话版：更好用、更体贴）

1. **托盘弹窗静默设置 (Tray Popup Settings)**：
   - 新增托盘弹窗全局开关，用户可以开启/关闭右下角或托盘区域的自动弹窗，避免在使用其他软件时受到高频干扰。
2. **WSL 网络网络专项设置**：
   - 针对 Windows WSL 用户的常见网络映射痛点，设计并实现了专有的 WSL 网络检测与设置面板，保证容器与主机的顺畅通信。
3. **Wails 服务绑定稳定性修复**：
   - 修复了后端 Wails 框架在服务绑定初始化时的边界时序 Bug，防止偶发性的前端 API 绑定失效导致页面空白的问题。

## 🛠️ 构建与开发工作流升级（大白话版：一键即成，绝无烦恼）

为了让团队伙伴更容易运行和编译本项目，我们特意编写了两个强大易用的 Shell 脚本：
1. **`dev.sh`（一键开发启动脚本）**：
   - **自动释放冲突端口**：启动开发服务器前，脚本会自动检测并强行释放占用 `9245` 端口的遗留进程，再也无需手动运行 `lsof` 和 `kill -9`，极大缩短开发启动调试时间。
   - **自动刷新绑定**：一键同步生成最新前端 Wails 绑定，即改即生效。
2. **`build.sh`（一键打包神器，支持交互与命令行）**：
   - **交互式菜单**：直接运行 `./build.sh` 会提供精美的终端彩色菜单，供你一键选择：
     1) 🚀 打包当前架构的生产版本 (`.app` 包) - 推荐
     2) 🌍 打包 Universal 通用版本 (`arm64` + `amd64`)
     3) 🛠️ 仅编译生产版本可执行文件
     4) ❌ 退出
   - **命令行静默打包**：完美支持 CI/CD 等自动化工作流，如 `./build.sh --current`、`./build.sh --universal`、`./build.sh --build`。
